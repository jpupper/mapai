Guía de Prompting para Vibe Coding

🎯 Prompting: El Arte de Comunicarte con la IA

El Vibe Coding es una nueva forma de programar donde la IA se convierte en tu copiloto de desarrollo. La clave del éxito está en saber comunicar tus ideas de manera efectiva. Aquí aprenderás las mejores prácticas para hacer prompts que generen código de calidad.

1

Sé Específico y Contextual

La IA necesita contexto para entender qué quieres lograr. No asumas que sabe todo sobre tu proyecto. Proporciona información clara sobre:

  • El lenguaje o framework que estás usando
  • El objetivo específico que quieres lograr
  • Las restricciones o requisitos técnicos
  • El estado actual de tu código

❌ Prompt Vago:

"Hazme una animación"

Problema: Demasiado genérico. La IA no sabe qué tipo de animación, en qué tecnología, ni para qué propósito.

✅ Prompt Específico:

"Crea una animación en p5.js de partículas que se muevan siguiendo el mouse. Las partículas deben tener colores aleatorios en tonos de rosa y morado, y dejar un rastro que se desvanezca gradualmente. El canvas debe ser de 800x600 píxeles."

Por qué funciona: Especifica la tecnología (p5.js), el comportamiento deseado, los colores, y las dimensiones.

📌 Puntos Clave:
  • Menciona siempre la tecnología o librería que usas
  • Describe el comportamiento esperado paso a paso
  • Incluye detalles visuales o técnicos importantes
2

Dos Enfoques: Incremental vs. Estructurado Completo

Existen dos estrategias principales para trabajar con IA en programación. La clave es saber cuándo usar cada una.

🔹 Enfoque A: Incremental (Divide y Conquista)

Cuándo usarlo: Proyectos exploratorios, cuando no tienes clara la arquitectura final, o cuando estás aprendiendo.

✅ Enfoque Incremental:

Paso 1: "Crea una escena básica en Three.js con un sistema de partículas 3D"

Paso 2: "Agrega controles de cámara OrbitControls a la escena"

Paso 3: "Crea una interfaz con un slider para controlar la velocidad de las partículas"

Paso 4: "Agrega sliders adicionales para tamaño y color"

Paso 5: "Implementa guardado de configuración en localStorage"

Ventajas: Cada paso es verificable y debuggeable. Fácil de corregir errores.

Desventajas: La IA va modificando código existente, lo que puede generar código menos optimizado y con "parches" que no se integran bien.

🔹 Enfoque B: Prompt Estructurado Completo

Cuándo usarlo: Cuando tienes clara la arquitectura completa, conoces bien los requisitos, y quieres código fresco y optimizado desde el inicio.

✅ Prompt Estructurado Completo:

"Crea una aplicación web completa con Three.js siguiendo esta estructura:

ARQUITECTURA:
- Sistema de partículas 3D con física básica
- Controles de cámara OrbitControls
- Panel de control con sliders (velocidad, tamaño, color)
- Sistema de guardado en localStorage
- Diseño responsive

REQUISITOS TÉCNICOS:
- Usar clases ES6 para organizar el código
- Separar lógica de renderizado y UI
- Implementar patrón Observer para los controles
- Optimizar para 60fps

ESTRUCTURA DE ARCHIVOS:
- index.html (estructura básica)
- main.js (inicialización)
- ParticleSystem.js (clase para partículas)
- UIController.js (clase para interfaz)
- style.css (estilos)"

Ventajas: Código fresco, bien estructurado y optimizado desde el inicio. La IA genera todo considerando la arquitectura completa.

Desventajas: Requiere que sepas exactamente qué quieres. Si algo falla, puede ser más difícil de debuggear.

⚠️ Importante: El enfoque incremental puede generar código de menor calidad porque la IA va "parcheando" sobre código existente. Cuando modifica código previo, a menudo quedan optimizaciones sin hacer y la estructura se vuelve menos coherente. Si conoces bien lo que necesitas, el prompt estructurado completo suele dar mejores resultados.
📌 Puntos Clave:
  • Enfoque Incremental: Mejor para exploración y aprendizaje
  • Enfoque Estructurado: Mejor para proyectos con arquitectura clara
  • Puedes combinar ambos: empieza con un prompt estructurado y luego itera
  • El código generado "fresh" suele ser de mayor calidad que el modificado iterativamente
3

Proporciona Ejemplos y Referencias

La IA aprende mejor cuando le muestras ejemplos de lo que quieres. Si tienes código existente, fragmentos de referencia, o incluso descripciones visuales, inclúyelos en tu prompt.

❌ Sin Referencia:

"Haz que el código se vea mejor"

Problema: "Mejor" es subjetivo. La IA no sabe qué aspecto específico mejorar.

✅ Con Referencia:

"Refactoriza este código para que use el patrón de diseño similar al ejemplo de la documentación oficial de p5.js. Aquí está mi código actual:

[tu código]

Quiero que las funciones estén mejor organizadas, con nombres descriptivos, y que uses constantes para los valores mágicos como colores y tamaños."

Por qué funciona: Proporciona el código actual, menciona un estándar de referencia, y especifica qué aspectos mejorar.

📌 Puntos Clave:
  • Comparte código existente cuando pidas modificaciones
  • Menciona ejemplos de la documentación oficial
  • Describe visualmente lo que quieres ("como un efecto de agua", "estilo minimalista")
  • Incluye enlaces a referencias visuales si es posible
4

Especifica el Formato de Salida

Dile a la IA exactamente cómo quieres que estructure la respuesta. ¿Quieres solo código? ¿Con explicaciones? ¿En archivos separados? Ser claro sobre esto ahorra tiempo.

❌ Sin Especificar Formato:

"Crea un shader en GLSL para un efecto de ondas"

Problema: No especifica si quiere el shader completo, solo el fragment shader, con o sin explicaciones, etc.

✅ Formato Especificado:

"Crea un shader en GLSL para un efecto de ondas. Necesito:
1. El código del vertex shader
2. El código del fragment shader
3. Comentarios explicando las partes clave
4. Los uniforms necesarios y sus tipos
5. Un ejemplo de cómo integrarlo en Three.js"

Por qué funciona: Lista exactamente qué componentes necesita y en qué orden.

📌 Puntos Clave:
  • Especifica si quieres código completo o solo fragmentos
  • Indica si necesitas comentarios o explicaciones
  • Menciona si quieres archivos separados (HTML, CSS, JS)
  • Pide ejemplos de uso si es código de librería
5

Itera y Refina: El Diálogo es Clave

El vibe coding no es un proceso de un solo paso. Es una conversación. Si el resultado no es exactamente lo que querías, no empieces de cero: refina el prompt basándote en lo que ya tienes.

❌ Empezar de Cero:

[La IA genera código con un error]

Usuario: "Olvida todo, hazlo de nuevo pero diferente"

Problema: Pierdes el progreso y la IA no aprende qué estuvo mal.

✅ Refinamiento Iterativo:

[La IA genera código con un error]

Usuario: "El código funciona bien, pero las partículas se mueven demasiado rápido. Reduce la velocidad a la mitad y agrega una opción para controlarla con un slider"

[Después de implementar]

Usuario: "Perfecto. Ahora agrega un efecto de fade-out cuando las partículas llegan al borde del canvas"

Por qué funciona: Construye sobre lo que funciona, hace ajustes específicos, y mantiene el contexto.

📌 Puntos Clave:
  • Identifica qué funciona y qué no en el código generado
  • Haz ajustes específicos en lugar de pedir todo de nuevo
  • Usa frases como "modifica esto", "ajusta aquello", "agrega esto"
  • Mantén el contexto de la conversación
  • Celebra los pequeños logros y construye sobre ellos

🚨 Plan de Emergencia: Cuando la IA Destruye Tu Código

A veces, la IA puede generar código que rompe completamente tu proyecto. ¡No entres en pánico! Si usas Git correctamente, siempre puedes volver atrás.

💣 Situación: La IA Falló y Destruyó Todo

Si la IA generó código que:

  • Rompió completamente tu aplicación
  • Eliminó funcionalidades importantes
  • Generó errores que no puedes resolver
  • Hizo cambios que no entiendes

✅ Solución: Resetea el Repositorio

Usa este comando para volver al último estado funcional que subiste a GitHub:

git reset --hard origin/main
⚠️ IMPORTANTE:
  • Este comando elimina TODOS los cambios locales que no hayas commiteado
  • Tu código volverá exactamente al estado del último git push
  • No hay forma de recuperar los cambios después de ejecutarlo
  • Úsalo solo cuando estés seguro de que quieres descartar todo

📋 Flujo de Trabajo Recomendado

# 1. Antes de pedirle a la IA cambios grandes, haz commit
git add .
git commit -m "Versión funcional antes de cambios"
git push origin main

# 2. Ahora pídele a la IA los cambios
# [La IA hace cambios...]

# 3a. Si funciona bien, haz commit de nuevo
git add .
git commit -m "Nuevas funcionalidades agregadas"
git push origin main

# 3b. Si la IA rompió todo, resetea
git reset --hard origin/main
# Ahora intenta con instrucciones diferentes
💡 Consejo Pro:

Haz commits frecuentes con mensajes descriptivos. Así siempre tendrás puntos de retorno seguros. Piensa en cada commit como un "punto de guardado" en un videojuego.

🔄 Alternativa: Ver Cambios Antes de Resetear

Si quieres revisar qué cambió antes de resetear todo:

# Ver qué archivos cambiaron
git status

# Ver los cambios específicos
git diff

# Si decides resetear
git reset --hard origin/main

🎓 Resumen: Las 5 Reglas de Oro del Vibe Coding

  1. Sé Específico y Contextual
    Proporciona toda la información necesaria: tecnología, objetivo, restricciones.
  2. Elige el Enfoque Correcto
    Incremental para explorar, Estructurado Completo para arquitectura clara.
  3. Proporciona Ejemplos
    Comparte código existente, referencias, y descripciones visuales.
  4. Especifica el Formato
    Define cómo quieres recibir el código: completo, comentado, separado.
  5. Itera y Refina
    Mantén la conversación, ajusta sobre lo que funciona, no empieces de cero.
💡 Consejo Final: El vibe coding es una habilidad que se desarrolla con la práctica. No te frustres si los primeros prompts no dan resultados perfectos. Experimenta, aprende de los errores, y descubre tu propio estilo de comunicación con la IA. ¡La magia está en el proceso!
